# Migration ‚Äî v2.5.0

Applied by `migrate-project` to projects whose `agents/.plugin-version` records `2.4.0` or earlier.
This is the large architectural release. Most deltas are additive; `agents/CLAUDE.md` is a full rewrite.

## Project-side deltas

### Files added

- **`agents/.session-type`** ‚Äî one-line file recording the session type (`claude-code`, `codex`, or `cowork`). Written by `init-project` on Phase 3; written by `migrate-project` on first migration run if missing.
  - Value to write: detect from current environment:
    - `${CLAUDE_PLUGIN_ROOT}` set and `${CODEX_PLUGIN_ROOT}` not set ‚Üí `claude-code`
    - `${CODEX_PLUGIN_ROOT}` set ‚Üí `codex`
    - Neither set ‚Üí `cowork`
  - Detection: if `agents/.session-type` already exists, skip (don't overwrite ‚Äî the marker was written at init time and reflects the project's primary session type).

- **`agents/SITUATIONS.md`** ‚Äî situation router table. Split out of root CLAUDE.md per the ¬ß7 Risk 1 decision (CLAUDE.md ‚â• 280 lines ‚Üí router moves to `agents/SITUATIONS.md`, one-line pointer stays in root CLAUDE.md).
  - Template source: `skills/init-project/references/phase-3-three-folder-create.md` ‚Üí `agents/SITUATIONS.md` template block.
  - Detection: if `agents/SITUATIONS.md` already exists and is non-empty, surface a diff against the template and propose merge ‚Äî never silent overwrite.
  - If `agents/SITUATIONS.md` does not exist: create it from the template.

- **`agents/.plugin-version`** ‚Äî created by `migrate-project` itself after all deltas are applied. Contains the current plugin version string (e.g. `2.5.0`). Not a pre-existing file; `migrate-project` writes it as the final step.

- **`agents/.plugin-version-skip`** ‚Äî per-version skip marker. Only created if the user explicitly chooses "skip and defer" at the drift-detector prompt. Contains the version string of the plugin version to skip. Re-prompts on the next plugin update (the skip resets because the new version string won't match). Not created by this migration ‚Äî created by user action only.

### Files modified

- **Root `CLAUDE.md`** ‚Äî full rewrite to the v2.5.0 "brain" structure.
  - New structure: Coding Standards ‚Üí Non-negotiable rules (rules 1‚Äì10) ‚Üí Skill index table ‚Üí Hooks index table ‚Üí Situation router pointer ‚Üí Folder map ‚Üí Pre-task classification ‚Üí Extended Context stub.
  - Rules added vs v2.4.0 template: rule #9 (hybrid shadcn-first component selection), rule #10 (use the skill index ‚Äî don't improvise workflows).
  - Hooks index table: new rows for `SessionStart (drift-detector)`, `PostToolUse (design-scan)`, `PostToolUse (design-check dispatcher)`.
  - Skill index table: new row for `migrate-project`.
  - Situation router line: now points to `agents/SITUATIONS.md` instead of inlining the table.
  - Template source: `skills/init-project/references/phase-3-three-folder-create.md` ‚Üí root `CLAUDE.md` template block (v2.5.0 version).
  - Detection: if root CLAUDE.md already contains the string `Use the skill index below. Don't improvise` (rule #10), this delta may already be applied ‚Äî surface a diff and let the user decide.
  - Customization check: diff the project's current root CLAUDE.md against the v2.4.0 template version. Any diverged lines are surfaced. User chooses: (a) overwrite with new template (preserving content under `## Extended Context`), (b) manual merge, (c) skip this delta.
  - **Preserve** any content the user has added under `## Extended Context` in the existing CLAUDE.md ‚Äî copy it into the new template's `## Extended Context` section.

- **`agents/CLAUDE.md`** (agents-tier CLAUDE.md, if present) ‚Äî not modified by this migration. The root `CLAUDE.md` is the brain in v2.5.0; `agents/CLAUDE.md` is not part of the v2.5.0 spec. If present, leave as-is and surface a note: "agents/CLAUDE.md exists ‚Äî it is not modified by this migration. The root CLAUDE.md is the always-loaded brain."

- **`agents/TOOLING.md`** ‚Äî re-ask package manager question and regenerate from the new ask-at-init template (Phase A4 change). This removes any personal path references that were in pre-v2.5.0 TOOLING.md templates.
  - Detection: if `agents/TOOLING.md` contains a home-directory path (`~/` or `/Users/`), this delta is needed.
  - Propose: "TOOLING.md contains a personal path. Regenerate from the v2.5.0 template (package manager: [detected from lockfile])? (y/skip)"
  - If yes: detect the existing package manager from lockfile (`bun.lockb` ‚Üí bun, `pnpm-lock.yaml` ‚Üí pnpm, `yarn.lock` ‚Üí yarn, `package-lock.json` ‚Üí npm), regenerate TOOLING.md from the template with the detected manager filled in.
  - If skip: leave as-is and add a note to the migration summary.

- **`agents/STRUCTURE.md`** ‚Äî add `Monorepo:` and `App paths:` fields if not present (Phase C monorepo detection work).
  - Detection: if `agents/STRUCTURE.md` exists and does NOT contain a `Monorepo:` field, this delta is needed.
  - Detection logic for monorepo: check for `pnpm-workspace.yaml`, `turbo.json`, or a `workspaces` field in root `package.json`.
  - Propose: "Add `Monorepo: true/false` and `App paths:` fields to STRUCTURE.md? (y/skip)"
  - If yes: append the fields at the top of the "Stack" or "Surfaces" section.
  - If `agents/STRUCTURE.md` does not exist: skip this delta ‚Äî STRUCTURE.md is created by `build-component` on first run.
