# Migration — v4.0.0

Applied by `migrate-project` to projects whose `brain/.plugin-version` records `3.x` or earlier.

The **design-engine** release. It adds an archetype-driven design rulebook, a calibration/moodboard step, a confidence-scored taste ledger, motion / image / build-order / done-checklist craft canon, and fixes the orchestration wire so UI chapters actually route through the design skills. It also fixes Codex skill discovery.

Most of the release is **plugin-side skill behavior** that takes effect on plugin update with no project file to migrate. The project-side deltas below are the files a `3.x` project should update.

---

## Project-side deltas

### [ADD] `brain/TASTE.md`

New confidence-scored learned-preferences ledger. Scaffold from `templates/TASTE.md` (replace `[Project Name]`). Safe to add to any project — it starts effectively empty and fills from real session signals. `save-session` writes it; `build-page` and `design-check` read it.

Detection: if `brain/TASTE.md` already exists, skip (it accumulates — never overwrite).

### [MODIFY] `brain/DESIGN.md` — new frontmatter fields + prose sections

The v4.0.0 DESIGN.md template adds to the frontmatter:

- **`archetype:`** — `dashboard | marketing | content | commerce | app`. The one classification that selects the design rulebook (component source, composition, motion class, era-sensitivity).
- **`motion:`** — register / duration bands / easing / rules.
- **`icon:`** — family / style.

…plus `## Archetype`, `## Motion`, and `## Icons` prose sections. For an existing project: surface the additions and **ask the user to set `archetype:`** (required — it drives downstream rules); fill `motion:` / `icon:` from the existing brand or mark `[VERIFY]`. **Do not overwrite existing token values** — only add the new blocks.

Detection: if `brain/DESIGN.md` already contains an `archetype:` field, this delta is applied — skip.

### [MODIFY] `brain/FUNDAMENTALS.md` — global refresh

`FUNDAMENTALS.md` is a global standard copied verbatim per project (never hand-edited). v4.0.0 adds four sections — `## Motion Spec`, `## Image Pipeline`, `## Build Order`, `## Definition of Done`. Refresh `brain/FUNDAMENTALS.md` from `templates/FUNDAMENTALS.md` (a clean copy is safe because it is never customized per project).

### [MODIFY] root `CLAUDE.md`

- **[RULE]** Rule 9 (shadcn-first) becomes **archetype-scoped**: shadcn-first for `dashboard` / `app`; bespoke expressive components for `marketing` / `content` (shadcn and community libraries are craft references, not installed aesthetics; use shadcn only for genuine primitives). Replace the rule-9 text with the v4.0.0 wording from `skills/init-project/references/phase-3-brain-create.md`.
- Add a **`calibrate`** row to the skill-index table (below `design-direction`).
- Add **`TASTE.md`** to the README file catalog (`brain/` section).

## Plugin-side changes (informational — no project-file delta)

- **New `calibrate` skill (27th).** The moodboard + FOLLOW/DEVIATE/REFUSE conventions-audit bridge inside `design-direction` (Phase 4 → 5), driving the Aside browser.
- **Behavior added** to `design-direction` (invokes calibrate, writes committed POV + anti-cliché), `build-page` (composition-variety rule, self-critique gate, reads TASTE), `build-component` (buy-vs-build by kind), `design-check` (reads + enforces TASTE), `save-session` (writes TASTE), and `ceo` / `worker` / `solo` (the orchestration wire — UI chapters route through the design skills; CEO refuses to merge UI that bypassed them).
- **Codex fix:** `.codex-plugin/plugin.json` gained `"skills": "./skills/"` + `"hooks": "./hooks/hooks.json"` pointers — Codex does not auto-discover a plugin's root `skills/`, so it was loading zero skills. `calibrate`'s `agents/openai.yaml` declares its Aside MCP dependency.

These take effect when the plugin updates; there is nothing to change in a project's files for them.
