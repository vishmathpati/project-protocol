# Migration — v3.0.0

Applied by `migrate-project` to projects whose `brain/.plugin-version` (or legacy `agents/.plugin-version`) records `2.5.0` or earlier.
This is the structural rebuild release. The three-folder layout (`cowork/`, `agents/`, `human/`) is replaced by a single `brain/` folder. Run the `migrate-to-brain` skill for the structural migration — it merges duplicates across the old folders, confirms before writing any file, and removes obsolete markers.

> **Note on v2.5.0 delta supersession:** the v2.5.0 migration created `agents/.session-type` (a one-line file recording `claude-code`, `codex`, or `cowork`). That marker no longer exists in v3.0.0 — author stamps (`· Cowork` / `· Claude Code` / `· Codex`) on canon files carry that signal instead. This migration must **not** re-create `agents/.session-type`. If it already exists, remove it as part of the migration.

## Project-side deltas

### Structural change: three folders → single `brain/`

The old layout had three tiers:

```
cowork/        ← orchestration tier
agents/        ← project canon tier
human/         ← steering tier (agenda.md)
```

The new layout has one:

```
brain/         ← single canon folder
```

Run `migrate-to-brain` to perform this migration. The skill:

1. Reads all files across `cowork/`, `agents/`, and `human/`.
2. For files that exist in more than one tier (e.g. `STATUS.md` in both `cowork/` and `agents/`), surfaces a diff and asks the user which version to keep or how to merge.
3. Confirms the full list of files to create, modify, and delete before writing anything.
4. Writes merged content to `brain/`.
5. Removes `cowork/`, `agents/`, and `human/` after confirmation.
6. Updates root `CLAUDE.md` folder map to point at `brain/`.

Detection: if `brain/` already exists and `agents/` does not, this structural delta is already applied — skip.

### Files removed

- **`agents/.session-type`** — superseded by author stamps on canon files. Must be deleted during migration; must not be re-created. (This file was introduced by the v2.5.0 migration. Its removal is a hard requirement of v3.0.0.)
- **`agents/SITUATIONS.md`** — merged into root `CLAUDE.md` situation router section or dropped if already superseded.
- **`cowork/`**, **`agents/`**, **`human/`** directories — removed after `brain/` migration is confirmed.

### Files modified

- **Root `CLAUDE.md`** — folder map updated from the three-tier layout to `brain/`. Skill index table updated with new skills (`ceo`, `worker`, `solo`, `git`, `grill`, `bug-fixing`, `handoff`, `migrate-to-brain`). Author-stamp rule replaces `.session-type` rule.
  - Detection: if root `CLAUDE.md` still references `agents/` or `cowork/` in its folder map, this delta is needed.
  - Customization check: diff the project's current root `CLAUDE.md` against the v2.5.0 template. Any content under `## Extended Context` is preserved into the new template.
  - Propose: "Update root CLAUDE.md folder map and skill index to v3.0.0? (y/skip)"

- **`brain/.plugin-version`** — written by `migrate-project` as the final step. Contains `3.0.0`. If `agents/.plugin-version` exists, it is replaced (not both kept).

### New skills (24 total, up from 16)

Eight skills are new in v3.0.0:

| Skill | Invocation | Description |
|---|---|---|
| `ceo` | user-invoked | CEO role: plan chapters, review Completion Reports, merge worker branches |
| `worker` | user-invoked | Worker role: claim a chapter, execute in a git worktree, write Completion Report |
| `solo` | user-invoked | Solo mode: single-agent execution without CEO/worker split |
| `handoff` | model-invoked | Worker teardown: write Completion Report, signal to CEO |
| `git` | user/model-invoked | Git operations gate: branch state, commit discipline, force-push guard |
| `grill` | user-invoked | Adversarial review: stress-test a plan or implementation |
| `bug-fixing` | user-invoked | Structured bug investigation: reproduce → isolate → fix → verify |
| `migrate-to-brain` | user-invoked (temporary) | One-time structural migration from three-folder to `brain/` |

### Author stamps replace `.session-type`

The `agents/.session-type` file (introduced in v2.5.0) indicated which session type initialized the project. In v3.0.0, this signal is carried by author stamps on canon file writes:

- `· Cowork` — written by a Cowork (Claude desktop) session
- `· Claude Code` — written by a Claude Code session
- `· Codex` — written by a Codex CLI session

Every significant write to `brain/BRIEF.md`, `brain/STATUS.md`, `brain/DISCOVERIES.md`, or `brain/CHANGELOG.md` must carry one of these stamps so the next agent can see who decided what.

`agents/.session-type` is deleted during migration and must not be re-created by any skill or hook. The v2.5.0 delta that created it is fully superseded.
