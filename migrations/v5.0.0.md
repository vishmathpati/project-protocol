# Migration — v5.0.0

The Project Protocol operating-system redesign. This manifest is extended alongside the v5 implementation and must be applied transactionally by the final v5 migration engine.

## Phase 1 — Core operating system

### [RENAME] Core skills

- `session-recap` → `recap`
- `audit-before-close` → `completion-check`
- `audit` → `project-audit`
- `discipline` → `change-check`
- `discussion-mode` → `discuss`

Update root CLAUDE routing and any project documentation that explicitly invokes the old names. Historical changelog text is not rewritten.

### [MODIFY] Universal Foundation

Replace the old init-project mega-flow with repository inspection plus universal questions only. Never re-interview an initialized project. Never create real chapters, fake history, brand/design direction, marketing, or research during Foundation.

Always ensure root CLAUDE and the core brain files exist: `.plugin-version`, BRIEF, STATUS, STRUCTURE, WORKLOG, CHANGELOG, agenda, WONT-DO, and chapter guide/template. Preserve customized content and show semantic consolidation proposals for human review.

### [MODIFY] Role and chapter ownership

Worker authority becomes chapter-scoped inside an isolated worktree. Workers may propose any chapter-required file change. The CEO owns approval, reconciliation, and integration. Existing restrictive worker instructions in project CLAUDE files must be replaced after showing the diff.

Chapter templates gain Why, Method, Done When, relevant canon/dependencies, constraints, carry-over, Completion Reports, and Verdicts. Do not rewrite existing chapter history.

### [MODIFY] Session canon

- STATUS keeps current state, active chapters, blockers, pending human decisions, and one Last saved session.
- WORKLOG becomes a temporary meaningful recovery buffer; preserve unresolved entries.
- BRIEF becomes the current universal project contract; semantic consolidation is manual.
- CHANGELOG receives accepted or shipped outcomes only.
- agenda becomes the next human checkout/role/chapter/decision view.

### [REMOVE] `SITUATIONS.md`

Migrate unique routing knowledge into concise root CLAUDE routing, verify it, then remove the file. Never delete it before reconciliation.

### Safety

Do not stamp `brain/.plugin-version` as 5.0.0 until every mandatory v5 delta validates. Do not use `git add -A`. Do not touch project code. Preserve customized canon and unrelated dirty files.
